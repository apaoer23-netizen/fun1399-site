## 五、SEO / EEAT / Information Gain 價值評估

### 評估標準

| 價值等級 | 標準 | 說明 |
|----------|------|------|
| **⭐⭐⭐⭐⭐ 極高** | 有完整對話截圖 + 金額 + 時間 + 過程 | 最稀有，無法複製 |
| **⭐⭐⭐⭐ 高** | 有過程描述 + 部分截圖 + 具體平台名稱 | 有 Information Gain |
| **⭐⭐⭐ 中** | 有過程描述但無截圖 / 或資訊較零散 | 可用但價值有限 |
| **⭐⭐ 低** | 僅有抱怨，無具體過程或證據 | 參考價值低 |
| **⭐ 極低** | 僅有一句「我被騙了」 | 無法使用 |

### 提升 SEO 價值的關鍵要素

助理在收集時，優先尋找包含以下要素的案例：

1. **具體數字**（金額、時間、次數）
2. **完整流程**（從接觸到被騙的步驟）
3. **原始證據**（對話截圖、轉帳紀錄）
4. **詐騙話術**（對方說了什麼引誘受害）
5. **平台資訊**（具體網址、LINE ID、帳號）
6. **後續發展**（是否報案、是否追回、目前狀態）

### 最強 SEO 價值的案例特徵

✅ **最佳案例模板**：
> 「2026 年 5 月，受害者透過 Facebook 廣告接觸到『XX 娛樂城』，對方聲稱『儲值 1000 送 5000』。受害者儲值 2 萬元後贏到 5 萬，申請提款時客服以『系統維護』拖延 2 週，之後要求『再儲值 3 萬解鎖提領功能』。受害者拒絕後帳號被凍結。損失 2 萬元，已報案 165。」

這類案例有：時間、金額、平台、話術、過程、結果 → **極高 Information Gain**

### 必須匿名化的項目

| 項目 | 處理方式 | 範例 |
|------|----------|------|
| **受害者姓名** | 改為「受害者 A」「投稿者」 | 王小明 → 受害者 A |
| **受害者聯絡方式** | 完全不記錄 | 手機、LINE、Email |
| **受害者帳號** | 完全不記錄 | 平台帳號、銀行帳號 |
| **具體金額（大額）** | 改為區間 | 250,000 → 20-30 萬 |
| **受害者照片** | 不使用 | 馬賽克或完全不收 |
| **LINE ID（部分）** | 中間碼隱藏 | abc123def → abc***def |
| **銀行帳號** | 只記末 4 碼 | ****5678 |

## 十、截圖整理規則

### 截圖標準

| 項目 | 要求 |
|------|------|
| **解析度** | 至少 1080p，文字清晰可讀 |
| **格式** | JPG（照片類）或 PNG（介面類） |
| **大小** | 單張不超過 5MB |
| **內容** | 需包含關鍵資訊（對話、時間、金額） |
| **去個資** | 受害者個資必須遮蓋 |

### 截圖處理步驟

1. **檢查內容**：確認截圖包含關鍵資訊
2. **去個資**：遮蓋受害者姓名、電話、帳號、照片
3. **標記重點**：用紅框/箭頭標出詐騙關鍵話術（可選）
4. **命名**：按命名規則重新命名
5. **分類**：放入對應資料夾
6. **記錄**：在 Google Sheet 填入截圖檔名

### 截圖品質檢查

❌ **不合格截圖**：
- 解析度太低，文字模糊
- 無關資訊過多，重點不明確
- 包含受害者個資未遮蓋

✅ **優秀截圖**：
- 清晰可讀，重點明確
- 有紅框標出詐騙話術
- 時間/金額清楚可見

## 十三、常見問題（FAQ）

**Q1：如果找不到新案例怎麼辦？**
A：沒有新案例就記錄「本日無新增」，不要硬找。品質 > 數量。

**Q2：如果案例只有抱怨沒有細節？**
A：記錄但標記為「低 SEO 價值」，不採用為主要案例。

**Q3：如果平台名稱無法確認？**
A：使用「平台-A」「平台-B」代號，或記錄網址。

**Q4：如果受害者不願提供截圖？**
A：尊重意願，只用文字描述，標記為「無截圖」。

**Q5：如果發現重複案例？**
A：比對案例編號，合併或標記「重複」。

**Q6：可以主動聯繫受害者索取更多資訊嗎？**
A：不建議。被動收集即可，避免干擾受害者。
