# fun1399.com 兩階段部署流程

## 🎯 部署策略

所有網站更新必須遵循「先預覽，後正式」的兩階段流程。

---

## 第一階段：Preview（預覽）

### 執行步驟

```
1. Build 網站
   ↓
2. 部署為「預覽版本」（draft / preview deploy）
   ↓
3. 提供預覽網址給用戶
   ↓
4. 等待用戶確認
```

### 預覽版本特性

- ✅ 獨立的 Preview URL（如：`https://xxx--fun1399.netlify.app`）
- ✅ 不影響正式網站 fun1399.com
- ✅ 可用於測試和審核
- ❌ 不會被搜尋引擎索引

### 回報內容

每次預覽部署後，請提供：
- 👉 **預覽網址**：`https://[preview-url]`
- 📋 **更新摘要**：改了什麼、為什麼改
- ⚠️ **注意事項**：需要特別檢查的地方

---

## 第二階段：正式部署

### 觸發條件

**必須等待用戶明確回覆「可以上線」**

### 用戶確認指令

用戶可以說：
- 「可以上線」
- 「發布」
- 「deploy 正式」
- 「確認無誤，發布」

### 執行步驟

```
1. 確認用戶已回覆「可以上線」
   ↓
2. Deploy 到正式網站 fun1399.com
   ↓
3. 驗證部署成功
   ↓
4. 回報部署完成
```

---

## ❌ 禁止事項

在以下情況，**絕對不要**部署到正式網站：

- 用戶尚未確認預覽版本
- 沒有提供預覽網址
- 自動化任務未經人工確認
- 任何緊急修復（即使是 bug 修復也要先預覽）

---

## 🔄 自動化任務調整

### SEO 內容自動化（週一/三/五）

**調整後流程：**

```
1. 分析現有文章
2. 選擇新主題
3. 生成文章
4. Build 網站
5. ⭐ 部署為 Preview 版本
6. ⭐ 發送預覽網址給用戶
7. ⭐ 等待用戶確認「可以上線」
8. ⭐ 部署到正式網站
9. 回報完成
```

---

## 🛠️ 技術實現

### Netlify Preview Deploy

```bash
# 部署預覽版本（draft）
netlify deploy --dir=build --draft

# 輸出 Preview URL
# https://xxx--fun1399.netlify.app
```

### 正式部署

```bash
# 用戶確認後才執行
netlify deploy --dir=build --prod
```

## 📋 執行檢查清單

### 每次部署前確認

- [ ] 是否已建立預覽版本？
- [ ] 是否已提供預覽網址？
- [ ] 用戶是否已確認「可以上線」？
- [ ] 是否確認要部署到正式網站？

### 禁止直接正式部署的情況

- [ ] 自動化任務生成內容
- [ ] 手動修改後未預覽
- [ ] 任何未經預覽的變更
