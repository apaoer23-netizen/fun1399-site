# Google Sheet 範本使用指南

## 一、文件說明

已準備 4 個 CSV 檔案，對應 Google Sheet 的 4 個工作表：

| 檔案名稱 | 工作表名稱 | 用途 |
|----------|-----------|------|
| sheet-cases.csv | Cases | 案例主表（10 筆示範資料） |
| sheet-fakeurls.csv | FakeURLs | 假網址資料庫（8 筆示範資料） |
| sheet-fakeline.csv | FakeLINE | 假 LINE ID 資料庫（8 筆示範資料） |
| sheet-dashboard.csv | Dashboard | 統計儀表板（含公式） |

## 二、建立 Google Sheet 步驟

### Step 1：建立新試算表
1. 前往 https://sheets.new
2. 建立空白試算表
3. 命名為「娛樂城防詐案例素材庫」

### Step 2：建立 Cases 工作表
1. 點擊「檔案」→「匯入」
2. 上傳 `sheet-cases.csv`
3. 匯入操作：選擇「取代目前工作表」
4. 工作表名稱改為「Cases」

### Step 3：建立 FakeURLs 工作表
1. 點擊左下角「+」新增工作表
2. 命名為「FakeURLs」
3. 「檔案」→「匯入」→ 上傳 `sheet-fakeurls.csv`
4. 匯入操作：選擇「取代目前工作表」

### Step 4：建立 FakeLINE 工作表
1. 新增工作表，命名為「FakeLINE」
2. 「檔案」→「匯入」→ 上傳 `sheet-fakeline.csv`
3. 匯入操作：選擇「取代目前工作表」

### Step 5：建立 Dashboard 工作表
1. 新增工作表，命名為「Dashboard」
2. 「檔案」→「匯入」→ 上傳 `sheet-dashboard.csv`
3. 匯入操作：選擇「取代目前工作表」
4. **重要**：公式欄位需要手動調整（見下方說明）

## 四、Dashboard 公式設定

Dashboard 工作表的公式需要根據實際工作表名稱調整。範本中的公式假設工作表名稱為英文。

### 需要手動確認的公式：

```
總案例數: =COUNTA(Cases!A:A)
本月新增: =COUNTIF(Cases!B:B,">="&DATE(YEAR(TODAY()),MONTH(TODAY()),1))
待審核數: =COUNTIF(Cases!R:R,"待審核")
已採用數: =COUNTIF(Cases!R:R,"已採用")
已發布數: =COUNTIF(Cases!R:R,"已發布")
高SEO價值案例: =COUNTIF(Cases!P:P,"極高")+COUNTIF(Cases!P:P,"高")
有報案案例: =COUNTIF(Cases!N:N,"是")
追回成功案例: =COUNTIF(Cases!O:O,"是")
TYPE-A數量: =COUNTIF(Cases!F:F,"TYPE-A")
假網址總數: =COUNTA(FakeURLs!A:A)
假LINE總數: =COUNTA(FakeLINE!A:A)
```

**注意**：如果工作表名稱有中文或空格，公式中需要用單引號包裹：
```
='Cases'!A:A
='FakeURLs'!G:G
```

## 五、條件式格式設定（視覺化管理）

讓重要資訊一眼可見：

### Cases 工作表

**P 欄（SEO 價值）**：
- 選取 P2:P1000
- 「格式」→「條件式格式設定」
- 規則 1：文字包含「極高」→ 綠色背景
- 規則 2：文字包含「高」→ 淺綠色背景
- 規則 3：文字包含「低」→ 淺紅色背景

**R 欄（處理狀態）**：
- 選取 R2:R1000
- 規則 1：文字包含「待審核」→ 黃色背景
- 規則 2：文字包含「已採用」→ 綠色背景
- 規則 3：文字包含「已發布」→ 藍色背景

**Q 欄（法律風險）**：
- 選取 Q2:Q1000
- 規則 1：文字包含「高」→ 紅色背景 + 白色文字
- 規則 2：文字包含「中」→ 橘色背景

## 六、分享設定

1. 點擊右上角「分享」
2. 設定權限：
   - 擁有者：你自己
   - 編輯者：助理（輸入助理的 Gmail）
   - 檢視者：需要查看的人
3. 建議啟用「通知我有人編輯時」

## 八、使用流程

### 每日流程：
1. 助理在各平台搜尋新案例
2. 發現案例 → 複製 Cases 工作表的最後一列，貼到新列
3. 填入所有必填欄位
4. 下載截圖 → 去個資 → 按命名規則存檔
5. 如有假網址 → 同時新增到 FakeURLs 工作表
6. 如有假 LINE ID → 同時新增到 FakeLINE 工作表
7. 檢查 Dashboard 統計（自動更新）

### 每週流程：
1. 主管審核「待審核」的案例
2. 標記「已採用」或「已存檔」
3. 檢查高法律風險案例
4. 確認匿名化是否完整
