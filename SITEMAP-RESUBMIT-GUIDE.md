# Sitemap 重新提交步驟
## 重要：URL 格式修改後必須執行

---

## 🚨 為什麼必須重新提交

**原因**：
- 原本的 Sitemap 包含 `.html` 結尾的 URL
- 現在 Sitemap 已改為無 `.html` 結尾
- Google 需要知道新的 URL 格式才能正確索引
- **不重新提交 = Google 繼續用舊格式索引 = 可能無法正確收錄**

---

## 📝 重新提交步驟

### 步驟 1：進入 GSC
1. 前往 https://search.google.com/search-console
2. 登入帳號
3. 選擇 fun1399.com 資源

### 步驟 2：移除舊 Sitemap
1. 左側選單點擊「Sitemap」（網站地圖）
2. 找到 `sitemap.xml`
3. 點擊該列右側的三個點 ⋮
4. 選擇「移除網站地圖」

### 步驟 3：重新提交
1. 在「新增網站地圖」欄位輸入：
   ```
   sitemap.xml
   ```
2. 點擊「提交」

### 步驟 4：確認成功
- 狀態應顯示「成功」或「處理中」
- 已發現網址：81 個

---

## ⏱️ 時間軸預期

| 時間 | 預期結果 |
|------|----------|
| **立即** | 狀態顯示「處理中」 |
| **1-3 天** | Google 開始抓取新 URL 格式 |
| **1-2 週** | 索引數量從 4 提升到 60+ |
| **對比** | 與 jc911blogs 相同格式 (380 索引) |

---

## ⚠️ 如果不重新提交的風險

- Google 繼續用舊的 `.html` URL 索引
- 可能導致重複內容問題
- 新文章可能無法正確收錄
- 浪費了今天修改 URL 格式的工作
