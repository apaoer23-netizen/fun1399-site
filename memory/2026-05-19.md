## 🔒 fun1399.com SSL 診斷（2026-05-19 19:55）

**問題**：Hans 報告 Chrome 偶發 `ERR_SSL_PROTOCOL_ERROR`

**診斷結果**：
- Netlify production deploy 正常（最後成功 deploy：2026-05-19 14:28）
- DNS 設定正確：A 記錄指向 Netlify load balancer（75.2.60.5 / 99.83.190.102）
- 關鍵發現：edge node `99.83.190.102` 對 `fun1399.com` 回傳空回應（EOF），但對 `www.fun1399.com` 正常 301
- 同一 IP、同一 port、同一請求，不同 hostname 有不同 SSL 行為 → **SSL 憑證部署故障**
- 測試結果：約 50% 請求成功，50% 失敗

**建議修復**：
1. 登入 Netlify → Site `statuesque-pithivier-bc9060` → Domain management → Renew certificate（強制續期）
2. 若無效，嘗試 Remove custom domain → 重新 Add custom domain
3. 檢查 DNS 是否有 AAAA 記錄，暫時移除避免 IPv6 優先

**影響**：正式用戶可能遇到約 50% 的訪問失敗率（視 DNS 解析到哪個 edge node 而定）。
